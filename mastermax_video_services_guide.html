1. تحليل التطبيق الحالي (Mastermax)

الخلاصة: التطبيق جاهز جزئياً لخدمات بث الفيديو المتقدمة، لكن يحتاج تحسينات محددة.

1.1 البنية التقنية الحالية

Framework: Flutter (Dart)
Video Player: video_player (^2.9.2) + chewie (^1.10.0)
Storage: Firebase Storage (حالياً)
Backend: Firebase (Firestore + Storage)
HTTP Client: dio (^5.4.0) + http (^1.3.0)

1.2 دعم بروتوكولات الفيديو

البروتوكول الحالة الإثبات
MP4 (Progressive) ✅ مدعوم video_player يدعم MP4 مباشرة
HLS (.m3u8) ✅ مدعوم video_player يدعم HLS تلقائياً (iOS/Android)
DASH ⚠️ غير مدعوم يحتاج مكتبة إضافية مثل better_player

1.4 Cloudflare Stream (موجود لكن غير مفعّل بالكامل)

ملاحظة: يوجد كود Cloudflare Stream في المشروع لكنه اختياري (Feature Flag).
الملف: lib/src/features/spotlight/services/cloudflare_stream_service.dart

1.5 نقاط القوة

  • يدعم HLS تلقائياً (مطلوب لـ Cloudflare Stream)
  • يستخدم dio للـ HTTP (مفيد للربط مع APIs)
  • بنية منظمة (Services + Providers)
  • Firebase موجود (يمكن استخدامه كـ Backend API)

1.6 نقاط تحتاج تحسين

  • لا يوجد Backend API منفصل (يعتمد على Firebase مباشرة)
  • لا يوجد نظام Transcoding (تحويل الفيديو)
  • لا يوجد Signed URLs (حماية الروابط)
  • لا يوجد تكامل مع Wasabi/Backblaze
  • لا يوجد تكامل مع Cloudflare CDN

2. الخدمات المناسبة لتطبيق Mastermax

التوصية: بناءً على تحليل التطبيق، Wasabi/Backblaze + Cloudflare CDN هو الخيار الأفضل لتطبيق Mastermax.

2.1 مقارنة سريعة

Cloudflare Stream

المميزات:

  • ✅ حل متكامل (تخزين + بث + تحويل)
  • ✅ سهل الربط (API مباشر)
  • ✅ يدعم HLS تلقائياً

العيوب:

  • ❌ مكلف جداً عند الانتشار الكبير
  • ❌ $1 لكل 1000 دقيقة مشاهدة
  • ❌ $5 لكل 1000 دقيقة تخزين

مناسب ل: MVP أو تطبيقات صغيرة

Wasabi/Backblaze + Cloudflare CDN

موصى به

المميزات:

  • ✅ أرخص بكثير (حوالي 500 ريال/شهر مقابل 228,750 ريال)
  • ✅ مرونة عالية في التحكم
  • ✅ Bandwidth Alliance (Egress مجاني/مخفض)
  • ✅ يدعم HLS (بعد Transcoding)

العيوب:

  • ⚠️ يحتاج Transcoding Server
  • ⚠️ يحتاج Backend API
  • ⚠️ إعداد أكثر تعقيداً

مناسب ل: تطبيقات كبيرة + انتشار عالي

2.2 لماذا Wasabi/Backblaze + Cloudflare CDN؟

التكلفة

فرق هائل في التكلفة: 500 ريال/شهر مقابل 228,750 ريال/شهر عند مليون مشاهدة يومياً

قابلية التوسع

يمكن التوسع بدون قلق من التكاليف المرتفعة. التكلفة تزيد تدريجياً مع الحجم

المرونة

تحكم كامل في جودة الفيديو، عدد النسخ، وإعدادات CDN

الأمان

يمكن تطبيق Signed URLs، Token-based Access، وCORS Policies

2.3 متى تستخدم Cloudflare Stream؟

  • إذا كان التطبيق في مرحلة MVP وعدد المشاهدات قليل (< 10,000 يومياً)
  • إذا لم يكن لديك فريق Backend قوي
  • إذا كنت تريد حل سريع بدون تعقيدات

4.2 تدفق العمل (من الرفع إلى المشاهدة)

1

رفع الفيديو

المستخدم يرفع الفيديو من التطبيق → Backend API

POST /api/videos/upload
2

التحويل (Transcoding)

Backend يرسل مهمة تحويل → Transcoding Service يحول إلى HLS

FFmpeg → .m3u8 + .ts segments
3

التخزين

الملفات المحولة تُرفع إلى Wasabi/Backblaze B2

PUT /b2api/v2/b2_upload_file
4

ربط CDN

Cloudflare CDN يقرأ من Wasabi/Backblaze عبر CNAME

CDN → B2 Bucket (Bandwidth Alliance)
5

المشاهدة

التطبيق يحصل على رابط HLS من API → video_player يشغله

GET /api/videos/{id}/stream → HLS URL

4.3 كود الربط في Flutter

أ. رفع الفيديو

// lib/src/features/spotlight/services/video_upload_service.dart
Future<String> uploadVideo(File videoFile) async {
  // 1. رفع إلى Backend API
  final formData = FormData.fromMap({
    'video': await MultipartFile.fromFile(videoFile.path),
    'title': title,
    'type': type,
  });
  
  final response = await dio.post(
    'https://api.mastermax.com/videos/upload',
    data: formData,
  );
  
  // 2. Backend يعيد videoId
  return response.data['videoId'];
}

ب. الحصول على رابط المشاهدة

// lib/src/features/spotlight/services/video_service.dart
Future<String> getVideoStreamUrl(String videoId) async {
  final response = await dio.get(
    'https://api.mastermax.com/videos/$videoId/stream',
    queryParameters: {
      'signed': 'true', // للحصول على Signed URL
    },
  );
  
  // Backend يعيد HLS URL من Cloudflare CDN
  return response.data['hlsUrl'];
  // مثال: https://cdn.mastermax.com/videos/abc123.m3u8
}

ج. تشغيل الفيديو

// lib/src/features/spotlight/widgets/video_player.dart
Future<void> _loadVideo(String videoId) async {
  // 1. الحصول على HLS URL
  final hlsUrl = await videoService.getVideoStreamUrl(videoId);
  
  // 2. تشغيل عبر video_player (يدعم HLS تلقائياً)
  _controller = VideoPlayerController.networkUrl(
    Uri.parse(hlsUrl), // .m3u8 URL
  );
  
  await _controller.initialize();
  _controller.play();
}

4.4 Backend API Endpoints المطلوبة

Endpoint الوظيفة المثال
POST /api/videos/upload رفع فيديو جديد استقبال ملف MP4 → إرسال لمهمة Transcoding
GET /api/videos/{id}/stream الحصول على رابط HLS إرجاع Signed URL من Cloudflare CDN
GET /api/videos/{id}/status حالة التحويل processing / ready / failed
DELETE /api/videos/{id} حذف فيديو حذف من B2 + Firestore

المعيار Cloudflare Stream Wasabi/Backblaze + Cloudflare CDN

7.1 التوصية النهائية

Wasabi/Backblaze + Cloudflare CDN

لماذا؟

  • ✅ التكلفة: أرخص بـ 226x عند الانتشار الكبير
  • ✅ قابلية التوسع: يمكن التوسع بدون قلق من التكاليف
  • ✅ المرونة: تحكم كامل في الجودة والإعدادات

ملاحظة: إذا كان التطبيق في مرحلة MVP وعدد المشاهدات قليل (< 10,000 يومياً)، يمكن البدء بـ Cloudflare Stream ثم الانتقال لـ Wasabi/Backblaze عند النمو.
