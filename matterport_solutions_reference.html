4. شرح الاستخدام العملي والسيناريوهات

4.1 ما هو Matterport Space؟

Space = مساحة واحدة تم تصويرها

كل Space هو نموذج 3D كامل لعقار واحد (شقة، فيلا، مكتب، متجر، إلخ).

مثال: إذا لديك 50 عقار، تحتاج 50 Active Spaces.

4.2 عدد Active Spaces لكل باقة:

الخطة عدد Active Spaces مثال عملي
Free 1 فقط عقار واحد فقط (للتجربة)
Starter 5-20 من 5 إلى 20 عقار نشط في نفس الوقت
Professional 20-150 من 20 إلى 150 عقار نشط
Business 100-300 من 100 إلى 300 عقار نشط
Enterprise Custom (مخصص) عدد غير محدود حسب الاتفاق
مهم جداً: Active Spaces = العقارات النشطة حالياً. يمكنك "أرشفة" (Archive) العقارات القديمة لتحرير مساحة للجديدة. الأرشفة مجانية، لكن إعادة تفعيلها قد يكلف $9.99 لكل Space.

4.3 سيناريو عملي: كيف يعمل Matterport مع تطبيق Mastermax؟

السيناريو: مستخدم يريد إضافة عقار جديد

  1. الخطوة 1: المستخدم يلتقط الصور

    المستخدم (مثلاً: مالك عقار أو موظف) يفتح تطبيق Mastermax ويختار "إضافة عقار جديد"

    التطبيق يطلب منه التقاط صور 360° للعقار (يمكن استخدام iPhone أو Android)

  2. الخطوة 2: رفع الصور إلى Matterport

    التطبيق يرفع الصور تلقائياً إلى Matterport عبر API

    Matterport يبدأ معالجة الصور وإنشاء Digital Twin (نموذج 3D)

    مدة المعالجة: عادة 30 دقيقة إلى ساعتين (حسب حجم العقار)

  3. الخطوة 3: Space جاهز

    بعد اكتمال المعالجة، Matterport يعيد Space ID للتطبيق

    التطبيق يحفظ Space ID في قاعدة البيانات

  4. الخطوة 4: عرض الجولة الافتراضية

    عندما يفتح مستخدم آخر العقار في التطبيق، يظهر زر "جولة افتراضية 3D"

    عند الضغط، التطبيق يفتح WebView مع رابط Matterport Embed URL

    المستخدم يستطيع التنقل داخل العقار بشكل تفاعلي 3D

4.4 شرح معنى "6 ساعات" و "24 ساعة" في Floor Plans:

ما هو Floor Plan؟

Floor Plan = مخطط أرضي (رسم هندسي يوضح توزيع الغرف والمساحات)

Matterport يمكنه إنشاء Floor Plan تلقائياً من Space الخاص بك.

Fast (24hrs) Floor Plans:

المعنى: Matterport يحتاج 24 ساعة لإنشاء Floor Plan

السعر: $19.99 لكل Floor Plan

متى تستخدمه: عندما لا تحتاج Floor Plan بشكل عاجل

مثال: أرسلت طلب Floor Plan اليوم الساعة 10 صباحاً → ستحصل عليه غداً الساعة 10 صباحاً

Express (6hrs) Floor Plans:

المعنى: Matterport يحتاج 6 ساعات فقط لإنشاء Floor Plan

السعر: $39.99 لكل Floor Plan (أغلى لأنها أسرع)

متى تستخدمه: عندما تحتاج Floor Plan بشكل عاجل

مثال: أرسلت طلب Floor Plan اليوم الساعة 10 صباحاً → ستحصل عليه اليوم الساعة 4 مساءً

Schematic Floor Plans (العادي):

المعنى: Floor Plan عادي بدون تحديد وقت محدد (قد يأخذ أيام)

السعر: $14.99 (Professional/Business) أو $19.99 (Starter)

متى تستخدمه: عندما لا تحتاج Floor Plan بشكل عاجل

ملاحظة مهمة: الساعات (6 أو 24) هي مدة معالجة Matterport لإنشاء Floor Plan، وليست مدة استخدام أو مشاهدة. بمجرد إنشاء Floor Plan، يمكنك استخدامه بدون حدود.

السيناريو الكامل من البداية للنهاية:

2

3

4.7 أسئلة شائعة:

هل عدد المشاهدات محدود؟

لا، عدد المشاهدات غير محدود. يمكن لأي عدد من المستخدمين مشاهدة Space بدون حدود.

ماذا لو تجاوزت عدد Active Spaces المسموح؟

يجب أرشفة بعض Spaces القديمة لتحرير مساحة، أو ترقية الخطة إلى باقة أعلى.

هل يمكن استخدام Space محدد بعدد معين من الساعات؟

لا، الساعات (6 أو 24) هي فقط لإنشاء Floor Plan، وليست لاستخدام Space نفسه. Space يمكن مشاهدته بدون حدود بعد إنشائه.

كم يستغرق إنشاء Space (Digital Twin)؟

عادة 30 دقيقة إلى ساعتين بعد رفع الصور، حسب حجم العقار وعدد الصور.

هل يمكن مشاركة Space مع أشخاص خارج التطبيق؟

نعم، يمكن مشاركة رابط Matterport مباشرة. الرابط يعمل على أي متصفح.

10.2 خطوات التكامل المقترحة:

  1. الخطوة 1: إنشاء حساب Matterport

  2. الخطوة 2: الحصول على API Credentials

  3. الخطوة 3: إضافة Dependencies

  4. الخطوة 4: إنشاء Service Layer

  5. الخطوة 5: تحديث UI

    تحديث PropertyVirtualTourScreen لدعم Matterport

10.3 مثال كود التكامل المقترح:

ملف: lib/services/matterport_service.dart

import 'package:dio/dio.dart';
import 'package:flutter_secure_storage/flutter_secure_storage.dart';

class MatterportService {
  final Dio _dio = Dio();
  final FlutterSecureStorage _storage = FlutterSecureStorage();
  static const String baseUrl = 'https://api.matterport.com/api/v1';
  
  Future<String> getAccessToken() async {
    final apiKey = await _storage.read(key: 'matterport_api_key');
    // Implementation for getting access token
    return apiKey ?? '';
  }
  
  Future<Map<String, dynamic>> getSpace(String spaceId) async {
    final token = await getAccessToken();
    final response = await _dio.get(
      '$baseUrl/spaces/$spaceId',
      options: Options(
        headers: {'Authorization': 'Bearer $token'},
      ),
    );
    return response.data;
  }
  
  Future<String> createSpace(Map<String, dynamic> spaceData) async {
    final token = await getAccessToken();
    final response = await _dio.post(
      '$baseUrl/spaces',
      data: spaceData,
      options: Options(
        headers: {'Authorization': 'Bearer $token'},
      ),
    );
    return response.data['id'];
  }
  
  String getEmbedUrl(String spaceId) {
    return 'https://my.matterport.com/show/?m=$spaceId';
  }
}
